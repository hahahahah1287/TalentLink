第三阶段：参数深度解析（面试官问你怎么训的，看这里）
这些参数不是随便填的，每一个都代表了架构师的考量。背下这几个核心概念，面试随便吹。
1. 为什么要用 finetuning_type: lora？
大白话：全量微调（Full Tuning）是把大模型脑子里的百亿神经元全改一遍，这需要几张 A100 显卡，Kaggle 根本跑不动。
技术解答：LoRA (Low-Rank Adaptation) 是低秩微调。它冻结了原模型的所有参数，只是在旁边加了一个很小的“外挂模块”（即矩阵 A 和 B）。训练时只更新这个外挂。这样显存占用降低了 90%，而且不会破坏模型原本的语言能力（防止灾难性遗忘）。
2. lora_target: all 是什么意思？
早期的 LoRA 只微调 Transformer 里的 Q 和 V 矩阵。
最佳实践：现在的业界共识是，微调 all（所有的线性层，包括 Q, K, V, O, Gate, Up, Down）效果最好，能让模型更全面地吸收你的意图分类知识。
3. 为什么设置 cutoff_len: 256？
截断长度。这代表模型一次处理的最大 Token 数。
意图识别的用户输入都很短（“帮我找工作”），加起来不到 100 个 Token。设置 256 既安全又能极大节省显存，加快训练速度。如果你设成 4096，其实后面 3000 多个位置都是补零（Padding），纯属浪费算力。
4. 神奇的组合：Batch Size 与 梯度累加
per_device_train_batch_size: 4 (显卡一次吃几条数据)
gradient_accumulation_steps: 4 (攒几次再更新参数)
技术解答：这两者相乘等于 真实 Batch Size = 16。Kaggle 的 T4 显存只有 16GB，如果 batch size 设为 16，直接 OOM（爆显存）。所以我让显卡一次吃 4 条，假装吃 4 次，然后在第 4 次的时候一起更新梯度。这是在贫民硬件上模拟土豪显卡的绝招。
5. 学习率：learning_rate: 2.0e-4
全量微调的学习率通常是 1e-5。
LoRA 黄金法则：因为 LoRA 参数量很少，必须要用更大的学习率（通常在 1e-4 到 3e-4 之间）才能让它学进去东西。2.0e-4 是经过无数人验证的 LoRA 甜点值。
6. 为什么跑 num_train_epochs: 10.0 这么多轮？
一般大模型训练只跑 1-3 轮。
但因为我们的数据集只有 100 条，非常小。如果不跑多几轮，模型根本记不住。10 轮能让 Loss 充分收敛。
